Glossary

  • static-typed (adjective): determining types at compile-time other than runtime execution
  • unsigned (adjective): only representing positive values
  • signed (adjective): representing both positive and negative values, with a median of 0 and -1
  • bit (noun): a binary number that can represent two states - on (1), and off (0)
  • utf8 (noun): an encoding format, presumably used on files that only need 256 unique characters to represent
  • utf16 (noun): another encoding format, presumably used on files that need 65536 unique characters to represent
  • floor (verb): to round a value by excluding the decimal point

Defining Variables

The first iconic command ever, is to define a variable. It's the most important, and one of the oldest commands, ever. You can define a variable by using the SET operator. Here are important things to know:

  • The identifier can only be represented with UPPERCASE, lowercase, and underscore (_) characters.
  • In the legacy version, variables can only be represented using an integer. In this version, you have to specify which type the value is, due to the now static-typed environment.
  • The value would represent a value that fits the provided type under other rules:
    • UINT8:
      • This value is an unsigned integer. It doesn't need to represent whether the value is negative or not.
      • The value can be represented between 0 - 255. If a value is provided beyond this range, the value will automatically be clamped into that range.
      • The value can only be represented using numbers. Attempting to use a decimal will floor the value.
      • The value takes up one byte per defined type, which is due to the value only needing eight bits (e.g. 8 bits = 1 byte).
      • You can use this value for when you want to represent a player's level, or the game's difficulty.
    • UINT16:
      • This value is an unsigned integer, like UINT8. It doesn't need to represent whether the value is negative or not.
      • The value can be represented between 0 - 65536. If a value is provided beyond this range, the value will automatically be clamped into that range, like UINT8.
      • The value can only be represented using numbers. Attempting to use a decimal will floor the value, like UINT8.
      • The value takes up two bytes per defined type, which is due to the value only needing sixteen bits.
      • You can use this value for when you want to represent the player's current XP (eXperience Points), or the player's current health/maximum health.
    • INT8:
      • This value is a signed integer. It shows whether the value is negative or not.
      • The value can be represented between -128 - 127. If a value is provided beyond this range, the value will automatically be clamped into that range, like UINT8.
      • The value can only be represented using numbers. Attempting to use a decimal will floor the value, like UINT8.
      • The value takes up one byte per defined type, which is due to the value only needing eight bits (e.g. 8 bits = 1 byte), like UINT8.
      • You can use this value for when you want to represent a simple number for mathematical equations, or the player's walking direction (if implemented in a specific way).
    • UFLOAT16:
      • This value is an unsigned float. It does not show whether the value is negative or not.
      • The value can be represented between 0 - 255.99609375. If a value is provided beyond this range, the value will automatically be clamped into that range, like UINT8.
      • The value can only be represented using numbers, and periods, like how this sentence is ended by. However, the compiler handles this in a unique way. Using a decimal will round the decimal to the nearest 0.00390625th, unlike how similar compilers would handle this.
      • The value takes up two bytes per defined type, which is due to the value only needing sixteen bits (e.g. 16 bits = 2 bytes), like UINT16.
      • You can use this value for when you want to represent an enemy's health, or when you want to represent a position of a paddle (like in Pong: The Classic Arcade Game).
    • UTF8STRING:
      • This value is a parameter-based type, which isn't a very common feature. A single parameter is needed, and the parameter is placed before the value. The parameter would make the compiler interpret the value in three different ways, depending on the parameter.
      • The value can be represented at any length - there are no constraints, or any limit to the length of the string.
      • The string uses the UTF8 encoding format, which makes it so the string can represent 256 unique characters.
      • The value takes up one byte per character, which the measuring can be dependent on how many characters are used in the string.
      • You can use this value for when you want to script a game dialogue, or a tutorial message of how to play the game.
      • The string value will throw an error if there is a character beyond the 256 character range. UTF16STRING will be a type that will later come in the future. (it was added 10/27/2024)
    • BOOLEAN:
      • This value is very simple; it represents two values - TRUE, or FALSE.
    • UTF16STRING:
      • This value is also a parameter-based type, which isn't a very common feature. A single parameter is needed, and the parameter is placed before the value. The parameter would make the compiler interpret the value in three different ways, depending on the parameter, like UTF8STRING.
      • The value can be represented at any length - there are no constraints, or any limit to the length of the string.
      • The string uses the UTF16 encoding format, which makes it so the string can represent 65536 unique characters.
      • The value takes up two bytes per character, which the measuring can be dependent on how many characters are used in the string.
      • You can use this value for when you would want to make messages compatible in different languages.
